# Rewrite Intake Checklist

Use this checklist before starting the RESEATO rebuild.

## Repository and access
- [ ] Source repository URL provided
- [ ] Branch strategy identified
- [ ] Build/run instructions confirmed

## Functional parity
- [ ] Pages/routes listed
- [ ] User flows documented
- [ ] API contracts noted
- [ ] Roles/permissions captured

## Technical baseline
- [ ] Current tech stack identified
- [ ] External integrations inventoried
- [ ] Database schema available
- [ ] Performance bottlenecks documented

## Migration controls
- [ ] Definition of done per module
- [ ] Test strategy selected (unit/integration/e2e)
- [ ] Rollout/cutover plan drafted
